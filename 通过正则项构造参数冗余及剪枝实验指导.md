## 概述

  

Vulcan 方法是一种****两阶段模型压缩技术****，通过以下步骤实现模型压缩：

  

1. ****训练阶段****：在 SFT（Supervised Fine-Tuning）过程中引入自定义的 Collapse Loss 正则项，促使 MLP 层的神经元权重趋向冗余（即多个神经元的权重向量趋于相同）

2. ****推理/剪枝阶段****：基于训练后的模型，通过分析激活值和权重相似度，将冗余神经元聚类，并执行剪枝

  

该方法的优势在于：

- 压缩后的模型****无需重新训练****即可保持较好的性能

- 通过正则项引导模型在训练过程中自动学习冗余结构

- 剪枝过程基于数据和激活感知，保留最重要的神经元

  

---

  

## 方法原理

  

### 核心思想

  

在 Transformer 架构中，MLP（Multi-Layer Perceptron）层通常包含大量的中间神经元（intermediate_size 通常为 hidden_size 的 4 倍）。Vulcan 方法的核心假设是：

  

****并非所有神经元都是独立的，许多神经元学习到了相似的特征表示，存在冗余。****

  

通过在训练过程中添加正则项，强制让部分神经元的权重向量趋于一致，从而在后续剪枝时可以合并这些冗余神经元而不显著影响模型性能。

  

### 数学公式

  

#### Collapse Loss 正则项

  

对于每一层 MLP，Collapse Loss 定义为：

  

```

L_collapse = λ1 * Σ||W_i - W_anchor||_1 + λ2 * Σ||W_i - W_anchor||_2^2

```

  

其中：

- `W_i` 表示第 i 个神经元的权重向量（拼接了 up_proj 和 gate_proj）

- `W_anchor` 表示该聚类中 anchor 神经元的权重向量

- `λ1` 和 `λ2` 是可学习的权重参数（初始化为 0，训练过程中自动调整）

- 第一项是 L1 正则，促使权重稀疏

- 第二项是 L2 正则，促使权重平滑收敛

  

#### 总损失函数

  

```

L_total = L_SFT + L_collapse

```

  

其中 `L_SFT` 是标准的 supervised fine-tuning 交叉熵损失。

## 实现流程

  

### 阶段一：训练时引入 Collapse Loss 正则项

  

#### 步骤 1.1：定义可学习参数

  

在模型初始化时，为模型添加两个可学习的参数 `lambda1` 和 `lambda2`：

  

```python

# 文件：src/llmtuner/train/sft/workflow.py

**if** finetuning_args.use_collapse_loss:

    model.lambda1 = torch.nn.Parameter(torch.tensor(0.0))

    model.lambda2 = torch.nn.Parameter(torch.tensor(0.0))

```

  

****为什么要可学习？****  

固定的 λ 值需要大量调参，而让模型在训练过程中自动学习这两个权重，可以自适应地平衡 SFT loss 和 collapse loss。

  

#### 步骤 1.2：在训练循环中添加 Collapse Loss

  

在每个训练 step 的 loss 计算中，添加 collapse loss：

  

```python

# 文件：src/llmtuner/train/sft/trainer.py

**if** self.finetuning_args.use_collapse_loss:

    **from** addons.collapse_loss **import** weight_collapse_loss

    unwrapped_model = self.accelerator.unwrap_model(model)

    lambda1 = unwrapped_model.lambda1

    lambda2 = unwrapped_model.lambda2

    loss_collapse = weight_collapse_loss(unwrapped_model, self.cluster_idx, lambda1, lambda2)

    loss += loss_collapse

```

  

****注意****：

- `cluster_idx` 需要提前计算（见阶段二）

- 在 DeepSpeed ZeRO-2 + gradient checkpointing 环境下，需要使用 weight proxy 和 backward hook 来避免梯度重复归约问题

  

#### 步骤 1.3：执行训练

  

使用标准的 SFT 训练脚本，只需添加 `--use_collapse_loss` 参数：

  

```bash

bash scripts/sft_ds_vulcan.sh

```

  

训练完成后，模型权重已经包含了冗余结构。

### 阶段二：基于激活值和权重聚类获取剪枝索引

  

#### 步骤 2.1：收集激活值

  

通过在小部分训练数据上 forward pass，统计每个神经元（down_proj 的输入）的平均激活强度：

  

```python

# 文件：src/addons/get_anchor.py

@torch.no_grad()

**def** get_activation(model, dataloader, save_path):

    activation = torch.zeros(num_hidden_layers, intermediate_size)

    **def** hook_fn(layer_idx):

        **def** fn(module, input, output):

            activation[layer_idx] += (input[0].sum(dim=(0,1))/batch_num).detach().cpu()

        **return** fn

    # 注册 hook 到 down_proj

    **for** layer_idx **in** range(num_hidden_layers):

        hooks.append(model.model.language_model.layers[layer_idx].mlp.down_proj.register_forward_hook(hook_fn(layer_idx)))

    # 遍历数据

    **for** batch **in** dataloader:

        output = model(**batch)

```

  

****关键点****：

- 只需要约 10,000 条样本即可

- 激活值用于后续选择 anchor neuron（选择激活最强的神经元）

  

#### 步骤 2.2：使用 Greedy Match 聚类（优先使用）

  

****Greedy Match 算法流程****：

  

```python

**def** get_cluster_greedy_match(activation, vectors, n_clusters):

    num_vectors = vectors.shape[0]  # 原始 intermediate_size

    cluster_size = num_vectors // n_clusters  # 每个聚类的大小

    # 计算所有权重向量之间的欧氏距离

    dist = euclidean_distances(vectors).astype(np.float32)

    assigned = np.zeros(num_vectors, dtype=bool)

    cluster_list = []

    **while** **not** assigned.all():

        unassigned = np.where(~assigned)[0]

        # 如果剩余不足一个聚类大小，全部归为一类

        **if** len(unassigned) <= cluster_size:

            # 选择激活最强的作为 anchor

            anchor_idx = unassigned[torch.argmax(activation[unassigned])].item()

            cluster_list.append({"anchor": anchor_idx, "neuron": unassigned.tolist()})

            **break**

        # 找到距离最近的一对神经元作为种子

        sub_dist = dist[np.ix_(unassigned, unassigned)]

        np.fill_diagonal(sub_dist, np.inf)

        nearest = sub_dist.min(axis=1)

        seed = unassigned[nearest.argmin()]

        # 找到与种子最近的 cluster_size-1 个神经元

        seed_dists = dist[seed, unassigned]

        nearest_local = np.argsort(seed_dists)[:cluster_size]

        members = unassigned[nearest_local]

        # 在该聚类中选择激活最强的作为 anchor

        anchor_idx = members[torch.argmax(activation[members])].item()

        cluster_list.append({"anchor": anchor_idx, "neuron": members.tolist()})

        assigned[members] = **True**

    **return** cluster_list

```

  

****为什么优先使用 Greedy Match？****

  

| 方法 | 优点 | 缺点 |

|------|------|------|

| ****Greedy Match**** | 保证每个聚类大小一致；计算高效；易于控制压缩率 | 可能不是全局最优 |

| K-Means | 全局优化 | 聚类大小不均衡，不利于并行计算 |

| Complete Linkage | 层次聚类，质量高 | 计算复杂度高 O(n³) |

  

****Greedy Match 的核心优势****：

1. ****确定性****：每次生成的聚类结果一致

2. ****均匀性****：每个聚类的神经元数量相同，便于硬件加速

3. ****高效性****：时间复杂度 O(n²)，适合大规模模型

  

#### 步骤 2.3：生成 cluster_idx 文件

  

对于每一层，生成聚类索引：

  

```python

# 指定每层的目标大小（例如压缩到原来的 1/2）

intermediate_size_list = [intermediate_size//2 **for** _ **in** range(num_hidden_layers)]

  

cluster_idx = [**None** **for** _ **in** range(num_hidden_layers)]

**for** l **in** range(num_hidden_layers):

    size = intermediate_size_list[l]

    **if** size == intermediate_size:

        **continue**  # 不压缩该层

    # 拼接 up_proj 和 gate_proj 的权重作为特征向量

    vectors = torch.cat([up_proj_w, gate_proj_w], dim=1).cpu().numpy()

    # 使用 greedy match 聚类

    cluster_idx[l] = get_cluster_greedy_match(activation[l], vectors, n_clusters=size)

  

# 保存为 JSON

json.dump(cluster_idx, open("cluster_idx_greedy_match.json", 'w'))

```

  

****cluster_idx 的数据结构****：

  

```json

[

  **null**,  // 第 0 层不压缩

  [

    {

      "anchor": 123,  // anchor 神经元的索引

      "neuron": [123, 456, 789, ...]  // 该聚类中所有神经元的索引

    },

    ...

  ],

  **null**,  // 第 2 层不压缩

  ...

]

```

### 阶段三：执行 MLP 剪枝

  

#### 步骤 3.1：替换 MLP 层

  

对于每一层需要压缩的 MLP：

  

```python

@torch.no_grad()

**def** pruning_mlp(model, cluster_idx):

    **for** l, layer **in** enumerate(model.model.language_model.layers):

        **if** cluster_idx[l] **is** **None**:

            **continue**

        mlp = layer.mlp

        config = copy.deepcopy(mlp.config)

        config.intermediate_size = len(cluster_idx[l])  # 新的中间层大小

        mlp_new = mlp.__class__(config).to(dtype=dtype).to(device)

        **for** i, c **in** enumerate(cluster_idx[l]):

            anchor_idx = c["anchor"]

            neuron_idxs = c["neuron"]

            # up_proj 和 gate_proj：直接复制 anchor 的权重

            mlp_new.up_proj.weight[i] = up_proj_w[anchor_idx]

            mlp_new.gate_proj.weight[i] = gate_proj_w[anchor_idx]

            # down_proj：对聚类中所有神经元的权重求和

            mlp_new.down_proj.weight[:, i] = down_proj_w[:, neuron_idxs].sum(dim=1)

        layer.mlp = mlp_new

    # 更新模型配置

    model.config.text_config.intermediate_size = intermediate_size

```

  

****剪枝逻辑解释****：

  

1. ****up_proj 和 gate_proj****：

   - 这两个层将输入从 hidden_size 映射到 intermediate_size

   - 由于聚类中的神经元权重已趋同，直接使用 anchor 的权重代表整个聚类

  

2. ****down_proj****：

   - 该层将 intermediate_size 映射回 hidden_size

   - 需要将聚类中所有神经元的贡献求和，因为它们在前向传播时都会被激活

## 关键代码解析

  

### Collapse Loss 实现

  

****完整代码****：`src/addons/collapse_loss.py`

  

#### 核心机制：Weight Proxy 与 Backward Hook

  

```python

**def** weight_collapse_loss(model, cluster_idx, lambda1, lambda2):

    **for** l **in** range(num_hidden_layers):

        **if** cluster_idx[l] **is** **None**:

            **continue**

        up_proj_w = model.model.language_model.layers[l].mlp.up_proj.weight

        gate_proj_w = model.model.language_model.layers[l].mlp.gate_proj.weight

        # 创建 weight proxy 避免 DeepSpeed ZeRO-2 的梯度重复归约问题

        weight_proxy = torch.cat([up_proj_w.detach(), gate_proj_w.detach()], dim=1)

        weight_proxy.requires_grad_(**True**)

        # 注册 backward hook，将 proxy 的梯度传递回原始权重

        weight_proxy.register_hook(_make_grad_hook(up_proj_w, gate_proj_w, up_size))

        # 计算聚类内神经元与 anchor 的权重差异

        **for** cluster **in** cluster_idx[l]:

            neuron_idxs = cluster["neuron"]

            anchor_idx = cluster["anchor"]

            diff_w = weight_proxy[neuron_idxs] - weight_proxy[anchor_idx]

            # L1 + L2 正则

            loss_cluster = lambda1 * diff_w.abs().sum() + lambda2 * diff_w.pow(2).sum()

            total_loss += loss_cluster

    **return** total_loss

```

  

****为什么需要 Weight Proxy？****

  

在 DeepSpeed ZeRO-2 + gradient checkpointing 环境下：

- 权重会被分片到多个 GPU

- gradient checkpointing 会在 backward 时重新计算 forward

- 直接使用 `torch.cat([up_proj_w, gate_proj_w])` 会导致梯度被重复归约

  

****解决方案****：

1. 使用 `.detach()` 创建独立的 tensor

2. 通过 `requires_grad_(True)` 使其成为新的计算图节点

3. 使用 backward hook 手动将梯度传递回原始参数

### Greedy Match 聚类算法（优先使用）

  

****完整代码****：`src/addons/get_anchor.py` 的 `get_cluster_greedy_match` 函数

  

#### 算法步骤详解

  

```

输入：vectors (权重矩阵), activation (激活值), n_clusters (目标聚类数)

输出：cluster_list (聚类索引列表)

  

1. 计算所有向量间的欧氏距离矩阵 dist[n, n]

2. 初始化 assigned 数组标记已分配的神经元

3. while 还有未分配的神经元:

   a. 在未分配的神经元中找到距离最近的一对 (seed, nearest)

   b. 以 seed 为中心，选择最近的 cluster_size 个神经元

   c. 在该聚类中选择激活值最大的作为 anchor

   d. 标记这些神经元为已分配

   e. 重复直到所有神经元都被分配

```

  

#### 可视化示例

  

假设有 8 个神经元，要压缩到 4 个（cluster_size=2）：

  

```

初始：[0, 1, 2, 3, 4, 5, 6, 7]

  

迭代 1:

- 找到最近的一对：(2, 5) 距离=0.3

- 以 2 为种子，选择最近的 2 个：[2, 5]

- 假设激活值：act[2]=0.8, act[5]=1.2 → anchor=5

- 聚类：{"anchor": 5, "neuron": [2, 5]}

  

迭代 2:

- 剩余：[0, 1, 3, 4, 6, 7]

- 找到最近的一对：(1, 4) 距离=0.4

- 聚类：{"anchor": 4, "neuron": [1, 4]}

  

... 直到所有神经元都被分配

```

## 注意事项

  

### 1. 训练阶段

  

- ****DeepSpeed 配置****：建议使用 ZeRO-2（`ds_zero2.json`），ZeRO-3 可能导致梯度同步问题

- ****Gradient Checkpointing****：如果启用，collapse loss 的 weight proxy 机制会自动处理兼容性问题

- ****可学习参数 λ****：初始化为 0 是安全的，模型会在训练初期自动学习合适的权重

- ****训练时长****：引入 collapse loss 不会显著增加训练时间（约增加 5-10%）

  

### 2. 聚类阶段

  

- ****样本数量****：10,000 条样本通常足够，太多会增加计算时间，太少会导致激活值统计不准确

- ****Greedy Match 优先****：除非有特殊需求，否则始终使用 `method="greedy_match"`

- ****分层压缩策略****：

  - 浅层（前 1/3）：建议不压缩或低压缩率（保留基础特征）

  - 中层：可以压缩 50%

  - 深层：可以压缩 50-75%（冗余度更高）

  

### 3. 剪枝阶段

  

- ****一次性操作****：剪枝是不可逆的，务必保存原始训练好的模型

- ****验证性能****：剪枝后建议在验证集上测试，确保性能下降在可接受范围内

- ****多模态模型****：如果使用 Processor（如 Qwen-VL），务必添加 `--use_processor` 参数

- ****tie_word_embeddings****：如果原始模型启用了此选项，保存时需保持一致

  

### 4. 常见问题

  

****Q1: 为什么 collapse loss 是 NaN？****

- 检查是否使用了 fp16，建议将 diff_w 转为 float32：`diff_w = diff_w.float()`

- 确保 cluster_idx 正确加载且不为 None

  

****Q2: 剪枝后模型大小没有减小？****

- 确认 `model.config.text_config.intermediate_size` 已正确更新

- 使用 safetensors 格式保存，避免共享权重导致的问题

  

****Q3: Greedy Match 太慢？****

- 对于超大模型，可以只对部分层聚类（设置 `cluster_idx[l] = None`）

- 距离矩阵计算是 O(n²)，可以考虑降采样或使用近似方法

  

****Q4: 如何选择合适的压缩率？****

- 建议从 50% 开始实验

- 在验证集上评估 perplexity 或任务指标

- 如果性能下降过大，降低压缩率（如 70%）

## 总结

  

Vulcan 方法通过****训练时正则化 + 推理时剪枝****的两阶段策略，实现了高效的模型压缩。其核心创新点在于：

  

1. ****可学习的损失权重****：λ1 和 λ2 自动平衡 SFT 和 collapse loss

2. ****Greedy Match 聚类****：保证均匀、确定性的聚类结果

3. ****DeepSpeed 兼容****：通过 weight proxy 和 backward hook 解决分布式训练问题

4. ****激活感知的 Anchor 选择****：保留激活最强的神经元作为代表

  

使用时只需在训练脚本中添加 `--use_collapse_loss`，训练完成后运行聚类脚本和剪枝脚本即可。整个过程无需修改模型架构或训练流程，具有良好的通用性和可扩展性。
