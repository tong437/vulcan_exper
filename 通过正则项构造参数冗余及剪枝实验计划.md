# 通过正则项构造参数冗余及剪枝实验计划

本文档基于仓库根目录的《通过正则项构造参数冗余及剪枝实验指导.md》和当前 LlamaFactory 代码结构整理。目标是在本地完成代码开发与小规模验证，再拉到服务器上用千问 3.5-0.8B 与医学 VQA 数据集 rad-vqa（parquet）完成训练、聚类、剪枝和评估。

## 1. 仓库观察结论

- 当前仓库包名是 `llamafactory`，实验指导中的 `llmtuner` 路径需要迁移到新版路径。
- 训练入口是 `src/train.py` 或 `llamafactory-cli train <config.yaml>`，最终进入 `src/llamafactory/train/tuner.py:run_exp()`。
- 默认 v0 SFT 路径为 `src/llamafactory/train/sft/workflow.py`，它负责加载 tokenizer、template、dataset、model，并创建 `CustomSeq2SeqTrainer`。
- SFT 损失注入点是 `src/llamafactory/train/sft/trainer.py:CustomSeq2SeqTrainer.compute_loss()`。
- 新训练参数应加入 `src/llamafactory/hparams/finetuning_args.py:FinetuningArguments`，YAML 配置会自动解析。
- 自定义 parquet 数据集可通过独立 `dataset_dir` 使用，只要该目录下有 `dataset_info.json`。LlamaFactory 已支持 `json/jsonl/csv/parquet/arrow/txt`。
- Qwen3.5 模板已经存在：`qwen3_5` 和 `qwen3_5_nothink`。如果 rad-vqa 是图像问答，应使用包含 `images` 列的 sharegpt 或 alpaca 多模态格式，并确保文本里的 `<image>` 数量与图片路径数量一致。
- 当前仓库没有 `src/addons/`、`collapse_loss.py`、`get_anchor.py`、`pruning.py` 这些指导文档里的文件，需要按当前工程风格新增。

## 2. 关键实验判断

指导文档的核心思想是两步：先用 Collapse Loss 让 MLP 神经元权重在指定簇内趋同，再基于激活与权重相似度做聚类和 MLP 剪枝。

实际落地时需要注意两点：

1. Collapse Loss 需要 `cluster_idx` 才知道哪些神经元应向哪个 anchor 靠拢，因此实验顺序应调整为“预聚类 -> 带正则 SFT -> 复算或复用聚类 -> 剪枝”。
2. 指导里把 `lambda1/lambda2` 设为可学习参数，但裸参数在梯度下降下可能变成负值或退化为 0。计划中先实现可配置的固定/调度系数作为主线，同时保留受约束的 learnable 版本用于对照实验。

本实验应使用 `finetuning_type: full` 作为主线。LoRA 会冻结基座 MLP 权重，无法直接构造 base MLP 神经元冗余；如需省显存，可另做 freeze-MLP 或 LoRA 对照，但不作为 Vulcan 主实验。

## 3. 数据准备计划

先不直接改仓库默认 `data/dataset_info.json`，而是新建独立数据目录，方便本地和服务器路径一致迁移。

建议结构：

```text
datasets/rad_vqa/
  dataset_info.json
  train.parquet
  validation.parquet
  images/
```

若 parquet 已包含 `messages` 和 `images` 列，采用 sharegpt 多模态格式：

```json
{
  "rad_vqa_train": {
    "file_name": "train.parquet",
    "formatting": "sharegpt",
    "columns": {
      "messages": "messages",
      "images": "images"
    },
    "tags": {
      "role_tag": "role",
      "content_tag": "content",
      "user_tag": "user",
      "assistant_tag": "assistant"
    }
  },
  "rad_vqa_val": {
    "file_name": "validation.parquet",
    "formatting": "sharegpt",
    "columns": {
      "messages": "messages",
      "images": "images"
    },
    "tags": {
      "role_tag": "role",
      "content_tag": "content",
      "user_tag": "user",
      "assistant_tag": "assistant"
    }
  }
}
```

若 parquet 是 `question/answer/image` 这类扁平列，先写一个转换脚本生成 LlamaFactory 标准列：

```json
{
  "messages": [
    {"role": "user", "content": "<image>问题文本"},
    {"role": "assistant", "content": "答案文本"}
  ],
  "images": ["images/xxx.png"]
}
```

本地先用 `max_samples: 16` 验证数据能被加载、图片路径能被解析、tokenization 不报错；服务器再放开完整训练。

## 4. 代码开发计划

### 4.1 新增 Vulcan 工具模块

建议新增目录：

```text
src/llamafactory/train/vulcan/
  __init__.py
  modeling.py
  collapse_loss.py
  clustering.py
  pruning.py
```

职责划分：

- `modeling.py`：识别通用 decoder 层路径和 MLP 的 `up_proj/gate_proj/down_proj`，兼容 Qwen/Qwen3.5/Qwen-VL 类模型。
- `collapse_loss.py`：实现簇内 anchor 约束，默认用 fp32 计算差异；需要兼容 ZeRO-2 与 gradient checkpointing。
- `clustering.py`：收集 down_proj 输入激活，拼接 `up_proj` 与 `gate_proj` 权重，生成 greedy match 聚类索引。
- `pruning.py`：按 cluster_idx 替换 MLP，`up_proj/gate_proj` 复制 anchor，`down_proj` 对簇内列求和，并更新 config。

### 4.2 接入训练参数

在 `FinetuningArguments` 中加入实验参数：

```yaml
use_collapse_loss: false
collapse_cluster_idx_path: null
collapse_lambda1: 0.0
collapse_lambda2: 0.0
collapse_learnable_lambda: false
collapse_log_steps: 10
```

后续可按需要补充：

```yaml
vulcan_keep_ratio: 0.5
vulcan_skip_first_ratio: 0.33
vulcan_activation_samples: 10000
vulcan_cluster_method: greedy_match
```

### 4.3 接入 SFT workflow 和 trainer

- 在 `run_sft()` 加载模型后，如果启用 `use_collapse_loss`，读取 `collapse_cluster_idx_path`。
- 如果启用 learnable lambda，给模型注册参数，并确保它们进入 optimizer。
- 在 `CustomSeq2SeqTrainer.compute_loss()` 中先计算标准 SFT loss，再追加 `weight_collapse_loss()`。
- 将 `loss_collapse/lambda1/lambda2` 通过 trainer log 暴露出来，便于在 TensorBoard、W&B 或 `trainer_log.jsonl` 中观察。

### 4.4 新增命令行脚本

建议放在 `scripts/vulcan/`：

```text
scripts/vulcan/collect_cluster_idx.py
scripts/vulcan/save_pruned_model.py
scripts/vulcan/eval_vqa_predictions.py
scripts/vulcan/inspect_model_redundancy.py
```

其中：

- `collect_cluster_idx.py`：加载模型与 rad-vqa 子集，收集激活并生成 `cluster_idx_greedy_match.json`。
- `save_pruned_model.py`：加载训练 checkpoint 与 cluster_idx，输出剪枝后的 HuggingFace 模型目录。
- `eval_vqa_predictions.py`：从 `generated_predictions.jsonl` 计算 VQA exact match、normalized exact match、token F1。
- `inspect_model_redundancy.py`：统计簇内 L1/L2 距离、cosine similarity，用于证明正则项确实构造了冗余。

## 5. 实验分组

建议至少跑四组，避免只比较“剪枝前后”而不知道正则项是否有效。

| 组别 | 训练 | 剪枝 | 用途 |
| --- | --- | --- | --- |
| A | 原始模型 zero-shot 或少量验证 | 否 | 了解基座在 rad-vqa 上的起点 |
| B | 标准 full SFT | 否 | 训练质量基线 |
| C | 标准 full SFT | 是 | 评估无 Collapse Loss 的普通剪枝损失 |
| D | Collapse full SFT | 是 | Vulcan 主实验 |

剪枝比例先从保守配置开始：

- `keep_ratio=0.75`：低风险 sanity check。
- `keep_ratio=0.50`：主实验。
- `keep_ratio=0.33`：压力测试，仅在 0.50 质量可接受后做。

分层策略：

- 前 1/3 层先不剪或只剪到 0.75，保留底层视觉/语言基础特征。
- 中后层先用 0.50。
- 如果 rad-vqa 对细粒度视觉信息敏感，优先放松浅层和跨模态相关层压缩率。

- 根据显存设置 `per_device_train_batch_size` 与 `gradient_accumulation_steps`。
- 设置 `use_collapse_loss: true` 和 `collapse_cluster_idx_path`。

## 7. 执行顺序

1. 本地检查 rad-vqa parquet schema，确定是标准 sharegpt 列还是需要转换。
2. 新建独立 `datasets/rad_vqa/dataset_info.json`，跑小样本 tokenization。
3. 实现 Vulcan 工具模块和单元测试。
4. 用 synthetic tiny MLP 测试 greedy match、collapse loss、剪枝等价性。
5. 用 Qwen3.5-0.8B 小样本跑 baseline SFT，确认训练入口、数据和模板正确。
6. 用未训练或 baseline 模型收集 1k-10k 样本激活，生成初始 `cluster_idx_greedy_match.json`。
7. 跑 Collapse full SFT，记录 SFT loss、collapse loss、簇内距离变化。
8. 对 baseline SFT 与 Collapse SFT 分别执行同压缩率剪枝。
9. 对未剪枝、普通剪枝、Vulcan 剪枝模型统一生成预测并计算 VQA 指标。
10. 汇总质量、参数量、checkpoint size、推理速度、显存占用和冗余统计。

## 8. 验证与验收标准

本地代码验收：

- `make quality` 或至少针对新增文件运行 ruff。
- `WANDB_DISABLED=true pytest -vv --import-mode=importlib tests/...` 覆盖新增 Vulcan 工具函数。
- 小样本 `llamafactory-cli train <debug.yaml>` 能完成至少一个 save/eval 周期。

服务器实验验收：

- Collapse SFT 的训练 loss 不出现 NaN。
- 簇内权重距离相比 baseline 明显下降。
- 剪枝后参数量和 checkpoint size 按目标比例下降。
- `keep_ratio=0.5` 下，Vulcan 剪枝模型优于“标准 SFT 后直接剪枝”的质量指标。
- 若 VQA exact match 下降过大，回退到 `keep_ratio=0.75` 或降低浅层剪枝力度。

## 9. 主要风险与处理

- 数据格式风险：rad-vqa 的 parquet schema 未知。先做 schema inspect 与 16 条样本加载，必要时写转换脚本。
- 模型类型风险：千问 3.5-0.8B 如果是纯文本模型，则不能直接消费图像；若 rad-vqa 含图像，需要确认使用的是带视觉塔的 Qwen3.5-VL/同类模型。
- 路径迁移风险：指导文档使用旧版 `llmtuner` 路径，代码实现必须以当前 `llamafactory` 为准。
- Lambda 退化风险：裸 learnable lambda 可能退化。主实验优先用固定/调度系数，learnable 作为对照。
- 分布式风险：优先 ZeRO-2，暂不把 ZeRO-3 作为主线；先在单卡/小样本确认逻辑。
- 剪枝保存风险：多模态模型需要同时保存 processor；保存后必须重新加载并跑一次生成。
